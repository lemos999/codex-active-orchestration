# Compact Prompt Composer

Write operating instructions as dense, direct prose with explicit action boundaries. Keep the always-on layer short, route detail into skills and references, and avoid repeating the same rule in several places. Preserve exact output obligations, safety boundaries, evidence requirements, and verification gates.

Prefer these transformations:

- Replace repeated examples with one general rule plus one sharp exception.
- Replace decorative explanation with observable behavior.
- Replace broad warnings with concrete checks.
- Replace long procedure blocks in the root contract with skill routing.
- Replace raw context dumps with locators, summaries, and expansion criteria.

Do not hide uncertainty, remove evidence obligations, or compress away acceptance criteria. Compactness is successful only when another agent can execute the same behavior with less irrelevant context.
